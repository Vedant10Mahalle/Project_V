# System Features and Functional Description

The Student Dropout Risk Prediction System is designed to assist academic administrators in identifying students who may be at risk of dropping out. The system uses machine learning techniques to analyze student data and predict academic outcomes.

The system provides an interactive web interface where administrators can enter student information, obtain predictions about student outcomes, and monitor risk levels.

The features of the system are designed to support both administrators and students by enabling early detection of academic risks and encouraging timely intervention.

## 1. Administrator Interaction with the System
The administrator plays a central role in operating the system. The administrator is responsible for entering student information, initiating predictions, and interpreting the results generated by the machine learning model.

**Step 1: Accessing the System**
The administrator accesses the system through the web application interface. After opening the system, the administrator is presented with the home page that provides an overview of the system and its purpose. From the home page, the administrator navigates to the dashboard where prediction and monitoring functions are available.

**Step 2: Entering Student Information**
The administrator enters the student’s details through the prediction form. The form contains several input fields that correspond to the features used by the machine learning model. The administrator provides information such as the student's demographic details, academic performance, and financial status.
The fields typically include:
- Age of the student
- Gender
- Admission grade
- First semester grade
- Second semester grade
- Scholarship status
- Debtor status
- Tuition fee payment status

**Step 3: Data Processing and Prediction**
After submission, the system sends the entered data to the backend server where it is processed. The backend system performs the following operations:
- The input data is converted into a numerical feature vector.
- The feature values are normalized using the previously saved scaler.
- The processed data is passed to the trained machine learning model.
- The model analyzes the input features and predicts the student’s academic outcome.
The predicted outcome can be one of the following categories:
- Dropout
- Enrolled
- Graduate

**Step 4: Displaying Prediction Results**
Once the prediction is generated, the backend server sends the result back to the web interface. 
The result page displays the following information to the administrator:
- Predicted academic outcome
- Risk level associated with the student
- Recommended actions for the administrator

The system maps prediction categories into risk levels for easier interpretation.
Example mapping:
- Dropout → High Risk
- Enrolled → Medium Risk
- Graduate → Low Risk

## 2. Student Perspective
The student does not directly interact with the prediction system in this implementation. Instead, the system operates as a decision-support tool for administrators and academic advisors. Based on the prediction results, administrators may take various actions that affect the student’s academic experience.

## 3. Academic Performance Features
Several features used in the system represent the academic performance of students.
- **Admission Grade**: Reflects the student’s academic preparedness.
- **First Semester Grade**: Reflects how well the student adapts to the academic environment.
- **Second Semester Grade**: Provides additional insight into the student’s continuous academic progress.

## 4. Demographic Features
Demographic features provide contextual information about students and may influence academic outcomes.
- **Age**: Reflects the student’s stage in life and external commitments.
- **Gender**: Included to capture possible patterns across different groups.

## 5. Financial Features
Financial factors often play a significant role in determining whether students can continue their education.
- **Scholarship Holder**: Financial support reduces stress and encourages persistence.
- **Debtor Status**: Indicates whether a student has outstanding financial obligations.
- **Tuition Fee Status**: Indicates whether the student has paid the required tuition fees.

## 6. Risk-Based Decision Support
The system categorizes students into different risk levels based on the prediction results. High-risk students are predicted to drop out, medium-risk need monitoring, and low-risk are expected to succeed.

## 7. Overall System Workflow
The complete system workflow follows several stages. The administrator accesses the system, inputs data, processes it via ML model, and takes action based on predicted risk level.
